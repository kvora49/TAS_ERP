Sales Bills (app/(dashboard)/sales/bills/) is the reference implementation for the repository/service/hook pattern. See its own module-level brain file (app/(dashboard)/sales/bills/brain.md) for full detail before replicating this pattern in another module. Do not duplicate GST/total calculation logic outside services/sales-bill.service.ts — that duplication was a confirmed, fixed bug, and the service layer is the single source of truth for all calculations. Updates to parent-child collections must be atomic — updateAtomic() calls a PostgreSQL function update_sales_bill_atomic via RPC (P0 item 7, P1 item 13), ensuring consistent rollback on failures.
